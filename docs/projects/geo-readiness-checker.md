# 기획안 — GEO Readiness Checker (AI 검색 인용 가능성 점수 도구)

> 가칭. URL을 붙여넣으면 **"이 페이지를 AI 검색(ChatGPT·Perplexity·Google AI Overview)이 인용할 가능성"을 0~100점**으로 매기고, 우선순위 수정안을 주는 무료 웹툴. 집계 데이터로 분기별 "State of GEO" 리포트를 발행해 권위를 쌓는다.

## 1. 목표 (끝나면 뭐가 달라지나)

- 누구나 자기 페이지 URL 하나로 **AI 인용 준비도 점수 + 처방**을 30초 안에 받는다.
- 도구가 SEO·AEO·바이럴을, 집계 리포트가 GEO 인용 권위를 가져오는 **복리 자산**을 1인+AI로 운영한다.

## 2. 왜 이 니치 (선정 근거)

플레이북의 5조건 + 요한의 불공정 우위 교집합에서 최상위:

| 조건 | 충족 방식 |
|---|---|
| 문제 즉시 해결 | "내 페이지가 AI에 인용될까?"는 지금 모두의 신생 고통 |
| 결과 공유 가능 | 점수 카드(OG 이미지·고유 URL) → 바이럴 루프 |
| 원본 데이터 보유 | 스캔 누적 → "산업별 평균 GEO 점수" = **GEO 인용 자석** |
| 롱테일 확장 | "[산업/CMS/페이지유형]을 위한 GEO 체크리스트" 프로그래매틱 수천 장 |
| 직답+스키마 | 도구 페이지 자체를 GEO 모범사례로 (메타-아이러니: GEO 잘 된 사이트가 GEO로 인용됨) |
| **불공정 우위** | 요한은 방금 GEO 리서치 완료 + AI 에이전트 운영자 → 도메인 신뢰 |

**핵심 feasibility:** 라이브로 LLM 엔진을 쿼리하지 않는다(비용·안티봇 회피). **페이지 정적 분석만**으로 점수 → MVP를 가볍게 1인+AI로 구현 가능.

## 3. 제품 정의

### 입력
URL 1개 (무료) → 점수카드 1장.

### 점수 축 (플레이북 §3과 1:1 매핑)
| # | 축 | 무엇을 검사 |
|---|---|---|
| 1 | 직답 구조 | 상단 TL;DR / 40~60단어 직답 / 질문형 H2·H3 |
| 2 | 스키마 | FAQPage·HowTo·Dataset·Article JSON-LD 유무·유효성 |
| 3 | 근거 밀도 | 통계·수치·인용구·외부 출처 링크 빈도 |
| 4 | 신선도 | 작성일·갱신일 노출, 최근성 |
| 5 | 엔티티 명확성 | 제목·정의·고유명사 구조, heading 위계 |
| 6 | 형식 적합 | 표/리스트 등 스니펫 친화 포맷, 가독 단락 |

→ 가중합 0~100 + 등급(A~F) + **우선순위 Top 3 수정 처방**(고친 예시 문구 포함).

### 공유 (바이럴 장치)
- 점수카드 OG 이미지 자동 생성 + 고유 URL → SNS·Slack 공유 시 점수가 미리보기로 노출.
- "친구 사이트와 비교" 듀얼 점수 → 비교 욕구 자극.

### 프로그래매틱 페이지 (SEO/AEO)
- `/checklist/[industry]`, `/checklist/[cms]`(WordPress·Webflow·Notion 등), `/checklist/[page-type]`(제품·블로그·랜딩) — 데이터셋 1개 → 페이지 수천.
- 각 페이지: TL;DR 직답 + FAQ 스키마 + 비교표 (플레이북 체크리스트 그대로 적용 = 자기 dogfooding).

### GEO 자산 (인용 권위)
- 누적 스캔 익명 집계 → 분기 **"State of GEO Readiness"** 리포트 (산업별 평균 점수·흔한 실패 Top10·개선 트렌드).
- 통계·인용구 풍부 → LLM 인용 자석. Reddit·LinkedIn·업계 매체에 시딩 → 브랜드 언급량↑.

## 4. 완료 기준 (MVP)

- [ ] URL 입력 → 6축 점수 + 등급 + Top3 처방 반환 (단일 페이지 분석)
- [ ] 점수카드 OG 이미지 + 공유 URL 생성
- [ ] 프로그래매틱 체크리스트 페이지 템플릿 1종 + 데이터 10건 시드
- [ ] 도구·체크리스트 페이지에 FAQPage/SoftwareApplication 스키마 적용 (self-dogfooding)
- [ ] 베이스라인: 핵심 질의 20개로 ChatGPT·Perplexity·Google AIO 인용 여부 기록

## 5. 작업 단계

| # | 단계 | 상태 | 메모 |
|---|------|------|------|
| 1 | 니치·범위 확정 + 도메인/네이밍 | 진행 | 본 기획안 = 1단계 산출물 |
| 2 | 점수 룰셋 v0 정의 (6축 가중치·판정 로직) | 대기 | 플레이북 §3 기반, 룰은 공개로 신뢰 확보 |
| 3 | MVP: URL 파서(HTML·JSON-LD 추출) + 점수 엔진 | 대기 | 정적 분석만, LLM 쿼리 없음 |
| 4 | 점수카드 UI + OG 이미지 + 공유 | 대기 | 바이럴 핵심 |
| 5 | 프로그래매틱 체크리스트 템플릿 + 시드 데이터 | 대기 | SEO 롱테일 |
| 6 | 스키마·AEO 셀프 적용 + 베이스라인 측정 | 대기 | 완료 기준 4·5번 |
| 7 | 분기 리포트 파이프라인(집계) 설계 | 대기 | GEO 자산, Phase 2 |

상태: `대기` → `진행` → `완료` / `취소`

## 6. 기술 스택 (가설)

- 프런트: Next.js (SSG/ISR로 프로그래매틱 페이지 + OG 이미지 라우트)
- 분석: 서버에서 대상 URL fetch → HTML/JSON-LD 파싱(cheerio 류) → 룰 엔진 점수화
- 저장: 스캔 결과 익명 집계 DB (리포트용)
- 배포: Vercel (이 세션 MCP에 `deploy_to_vercel` 있음 — 추후 활용 가능)
- ⚠️ 일부 사이트는 봇 차단(403)·JS 렌더 → 헤드리스 렌더 폴백 필요 여부는 3단계에서 판단.

## 7. 수익화 (나중)

- 무료: 단건 URL 스캔.
- 유료: 사이트 전체 크롤 스캔 + 변화 모니터링/알림 구독, 화이트라벨 리포트, 경쟁사 비교 추적.

## 8. 리스크 / 미검증

- `[가설]` 점수 룰의 "AI 인용 가능성" 예측력 — 실제 인용 데이터와의 상관은 베이스라인(6단계) 이후 검증 필요. 과대약속 금지.
- 경쟁: 기존 GEO/AEO 감사 도구 존재 가능 → 2단계 전 **경쟁 스캔**으로 차별점(공개 룰셋·점수카드 바이럴·1인 운영 투명성) 확정.
- 봇 차단으로 분석 실패하는 URL 처리 UX.

## 9. 결정 사항

- 라이브 LLM 엔진 쿼리 대신 **정적 분석**으로 MVP — 비용·안티봇·속도 때문. (`memory/decisions/2026-06-15-1138-geo-checker-niche.md`)
- 본 기획안은 `status: draft` — 다음 세션에서 2단계(룰셋 v0) 착수 전 경쟁 스캔 1회.
