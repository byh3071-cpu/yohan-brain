## A-1. 보고서 요약 템플릿

**카테고리:** 업무 문서 / **난이도:** ★

```
<context>경영 컨설팅 업무 환경</context>

아래 보고서를 읽고 다음 형식으로 요약해주세요:

1. 핵심 결론 (1문장)
2. 주요 근거 (3개, 각 1문장)
3. 의사결정자를 위한 권고사항 (1~2문장)

<report>
[보고서 전문 붙여넣기]
</report>

요약은 500자 이내로 작성하세요.
전문 용어는 괄호 안에 쉬운 설명을 덧붙이세요.
```

**사용법:** \[보고서 전문 붙여넣기\] 부분에 실제 보고서를 넣으면 돼요. 긴 보고서도 한 페이지 분량으로 압축돼요.

**예상 결과:** 핵심 결론, 근거 3개, 권고사항이 구조화된 500자 이내 요약문이 나와요.

```

다음 조건으로 업무 메일을 작성해주세요:

- 수신자: [직급/관계, 예: 팀장님, 거래처 담당자]
- 목적: [메일의 목적, 예: 프로젝트 일정 변경 요청]
```

**예상 결과:** 4단 구조의 깔끔한 업무 메일 초안이 나와요.

## A-4. 기획서 초안 템플릿

**카테고리:** 업무 문서 / **난이도:** ★★

```
<context>사업 기획 환경</context>

다음 정보를 바탕으로 기획서 초안을 작성해주세요:

- 프로젝트명: [프로젝트 이름]
- 배경: [왜 이 프로젝트가 필요한지 2~3문장]
- 목표: [달성하려는 구체적 목표]
- 대상: [누구를 위한 것인지]
- 예산 범위: [있으면 기재, 없으면 "미정"]
- 일정: [희망 기간]

기획서 구조:
1. 프로젝트 개요 (배경 + 목적, 반 페이지)
2. 현황 분석 (문제점 3개, 기회 요인 2개)
3. 추진 전략 (핵심 전략 2~3개, 각각 구체적 실행 방안 포함)
4. 기대 효과 (정량/정성 각 2개)
5. 일정 및 예산 (단계별 마일스톤)
6. 리스크 및 대응 방안 (주요 리스크 2~3개)

각 섹션은 구체적인 수치나 사례를 포함해주세요.
```

**사용법:** 프로젝트 기본 정보만 채우면 6개 섹션의 기획서 뼈대가 나와요. 이후 각 섹션을 대화를 통해 구체화하면 돼요.

**예상 결과:** 6개 섹션으로 구성된 2~3페이지 분량의 기획서 초안이 나와요.


**예상 결과:** 제목 3안 + 도입 + 소제목별 본문 + 결론이 포함된 초안이 나와요.

## B-3. 경쟁사 분석

```
<context>시장 분석 환경</context>
```

**예상 결과:** 비교 매트릭스와 차별화 기회가 포함된 경쟁 분석 보고서가 나와요.

**예상 결과:** 7개 항목이 채워진 구체적인 고객 페르소나 2개가 나와요.

## E-1. Chain-of-Thought (Wei 2022 패턴)

**카테고리:** 고급 기법 / **난이도:** ★★★

Wei 등(2022)이 제안한 Few-shot CoT 패턴이에요. 프롬프트에 "단계별 풀이 + 최종 답" 형태의 예시를 넣으면, 충분히 큰 모델이 같은 패턴으로 중간 추론 단계를 생성하며 정확도가 크게 올라가요. GSM8K에서 여덟 개 예시만으로 당시 최고 성능을 달성했어요.

```
다음 문제를 단계별로 풀어주세요.

[예시 1]
Q: 카페에서 커피 한 잔에 4,500원이에요. 3잔을 사고
   10% 할인 쿠폰을 사용하면 얼마를 내야 하나요?
A: 단계별로 풀어볼게요.
   1단계: 커피 3잔의 원래 가격 = 4,500 x 3 = 13,500원
   2단계: 10% 할인 금액 = 13,500 x 0.1 = 1,350원
   3단계: 최종 결제 금액 = 13,500 - 1,350 = 12,150원
   답: 12,150원

[예시 2]
Q: 팀원 8명에게 간식을 사려고 해요. 빵 1개에 3,000원,
   음료 1개에 2,000원이에요. 전원에게 둘 다 사주면
   총 비용은 얼마인가요?
A: 단계별로 풀어볼게요.
   1단계: 빵 비용 = 3,000 x 8 = 24,000원
   2단계: 음료 비용 = 2,000 x 8 = 16,000원
   3단계: 총 비용 = 24,000 + 16,000 = 40,000원
   답: 40,000원

이제 아래 문제를 같은 방식으로 풀어주세요:
Q: [실제 문제]
A:
```

**사용법:** \[실제 문제\] 부분에 풀려는 문제를 넣으면 돼요. 예시의 풀이 과정이 정확해야 해요, 틀린 풀이를 예시로 넣으면 모델이 그 오류를 그대로 모방해요 (Wei 2022). 단순 사실 질문에는 사용하지 마세요. 답이 뻔한 질문에 단계 생성을 강제하면 비용만 올라가요.

**예상 결과:** 예시와 같은 형태의 단계별 추론 과정 + 최종 답이 나와요.

## E-2. Self-Consistency (Wang 2022 패턴)

**카테고리:** 고급 기법 / **난이도:** ★★★

Wang 등(2022)이 제안한 Self-Consistency 기법이에요. 핵심은 greedy 디코딩(하나의 답) 대신, 다양한 추론 경로를 여러 개 샘플링하고 최빈(가장 일관된) 답을 선택하는 것이에요. CoT 위에 거의 추가 비용 없이 쌓이는 개선으로, GSM8K에서 +17.9%, SVAMP에서 +11.0%의 정확도 향상을 보였어요.

```
다음 문제에 대해 서로 다른 접근 방식으로 3번 풀어주세요.
각 풀이는 독립적으로 진행하세요.

문제: [복잡한 분석/판단 문제]

<approach_1>
[첫 번째 접근: 처음 떠오르는 자연스러운 방식으로]
결론:
</approach_1>

<approach_2>
[두 번째 접근: 다른 관점이나 다른 순서로]
결론:
</approach_2>

<approach_3>
[세 번째 접근: 또 다른 방식으로]
결론:
</approach_3>

<final_answer>
세 접근의 결론을 비교하세요.
- 일치하는 결론: [무엇이 일치하는지]
- 불일치하는 부분: [있으면, 왜 다른지]
- 최종 답: [가장 일관된 결론을 선택하고 근거 제시]
</final_answer>
```

**사용법:** 정답이 하나로 수렴하는 분석/판단 문제에 적합해요. 창작처럼 정답이 없는 과제에는 적합하지 않아요. 접근 수를 늘리면 정확도가 올라가지만 토큰 비용이 선형으로 증가하니 예산을 고려해야 해요.

**예상 결과:** 3가지 독립 분석 + 비교 + 최종 답이 나와요.

## E-3. Tree of Thoughts (Yao 2023 패턴)

**카테고리:** 고급 기법 / **난이도:** ★★★

Yao 등(2023)이 제안한 Tree of Thoughts 패턴이에요. 사고를 트리 노드로 보고, 모델이 자기 상태를 평가하며 전진하거나 백트래킹하게 만들어요. Game of 24에서 GPT-4의 CoT 성공률 4%를 ToT로 74%까지 끌어올렸어요.

```
다음 문제를 트리 탐색 방식으로 풀어주세요.

문제: [탐색이 필요한 복잡한 문제]

각 단계에서:
1. 가능한 다음 행동 2~3개를 나열하세요
2. 각 행동의 전망을 평가하세요 (유망/불확실/막다른길)
3. 가장 유망한 경로를 선택해 진행하세요
4. 막다른길에 도달하면 이전 단계로 돌아가 다른 경로를 시도하세요

탐색 기록:
[1단계]
- 선택지 A: [설명] -> 전망: [유망/불확실/막다른길]
- 선택지 B: [설명] -> 전망: [유망/불확실/막다른길]
- 선택지 C: [설명] -> 전망: [유망/불확실/막다른길]
=> 선택: [선택한 것과 이유]

[2단계]
(같은 구조 반복)

[막다른길 도달 시]
=> 백트래킹: [N단계]로 돌아가서 [다른 선택지] 시도

[최종 결과]
- 탐색한 경로 수: [N개]
- 최적 경로: [단계별 요약]
- 답: [최종 답]
```

**사용법:** 퍼즐, 전략 수립, 복잡한 의사결정처럼 탐색과 백트래킹이 필요한 과제에 적합해요. 단순 질의응답에는 과잉 도구예요.

**예상 결과:** 트리 형태의 탐색 기록 + 백트래킹 과정 + 최적 경로 + 최종 답이 나와요.

## E-4. Meta-Prompting (Suzgun 2024 Conductor-Expert 패턴)

**카테고리:** 고급 기법 / **난이도:** ★★★

Suzgun & Kalai(2024)가 정식화한 Meta-Prompting의 Conductor-Expert 패턴이에요. 하나의 LLM이 지휘자(Conductor) 역할을 하면서 과제를 서브태스크로 분해하고, 각 서브태스크에 전문가(Expert) 인스턴스를 할당하고, 출력을 통합하고 검증해요. GPT-4 기준 standard prompting 대비 +17.1% 개선을 보였어요. 태스크 불가지론적 스캐폴드이므로 새 도메인에 재튜닝 비용이 낮아요.

```
<context>복합 과제 분해 및 통합 환경</context>
아래 과제를 서브태스크로 분해하고,
각 서브태스크를 전문가에게 위임하고,
결과를 통합하세요.

과제: [복합적인 과제]

[1단계: 분해]
이 과제를 독립적인 서브태스크로 나누세요:
- 서브태스크 1: [설명] -> 필요한 전문성: [분야]
- 서브태스크 2: [설명] -> 필요한 전문성: [분야]
- 서브태스크 3: [설명] -> 필요한 전문성: [분야]

[2단계: 전문가 실행]

<expert role="[전문가 1의 역할]">
서브태스크 1에 대한 전문가 분석/실행:
[결과]
</expert>

<expert role="[전문가 2의 역할]">
서브태스크 2에 대한 전문가 분석/실행:
[결과]
</expert>

<expert role="[전문가 3의 역할]">
서브태스크 3에 대한 전문가 분석/실행:
[결과]
</expert>

[3단계: 통합 및 검증]
지휘자로서 각 전문가의 출력을 검토하세요:
- 전문가 간 모순이 있는가?
- 누락된 관점이 있는가?
- 검증이 필요한 사실이 있는가?

[최종 결과]
통합된 답변:
```

**사용법:** 여러 분야의 전문성이 필요한 복합 과제에 적합해요. 간단한 태스크에 과잉 적용하면 지휘자/전문가 호출 수만큼 토큰과 지연이 늘어나고 이득은 없어요 (Suzgun 2024). 전문가 역할을 "세계 최고 수학자" 식으로 과포장하지 마세요. 역할은 톤과 범위 지정 정도로 절제하는 것이 원논문의 권고예요. 지휘자가 전문가 출력을 무비판적으로 합치면 오류가 증폭되니, 3단계 검증을 생략하지 마세요.

**예상 결과:** 서브태스크 분해 + 전문가별 결과 + 교차 검증 + 통합 답변이 나와요.

## E-5. 구조화 출력 (XML 태그 + JSON 스키마)

**카테고리:** 고급 기법 / **난이도:** ★★★

Anthropic 공식 가이드가 권장하는 구조화 출력 패턴이에요. XML 태그로 입력 영역을 분리하고, JSON 스키마로 출력 형식을 강제하면 모델의 파싱 정확도가 올라가고 다운스트림 도구 연동이 안정화돼요. 구조화 출력은 모델의 자유로운 생성을 제한하는 것이 아니라, 품질을 높이는 가드레일 역할을 해요.

```
<instructions>
<context>제품 리뷰 분석 시스템 환경</context>
아래 리뷰들을 분석하여 지정된 JSON 형식으로 출력하세요.
</instructions>

<reviews>
[리뷰 1~N 붙여넣기]
</reviews>

<output_schema>
다음 JSON 형식을 정확히 따르세요:
{
  "summary": {
    "total_reviews": (정수),
    "average_sentiment": ("긍정" | "중립" | "부정"),
    "confidence": (0.0~1.0)
  },
  "top_positives": [
    {"theme": "(주제)", "count": (언급 횟수), "example": "(대표 인용)"}
  ],
  "top_negatives": [
    {"theme": "(주제)", "count": (언급 횟수), "example": "(대표 인용)"}
  ],
  "actionable_insights": [
    {"priority": ("높음"|"중간"|"낮음"), "suggestion": "(구체적 개선안)"}
  ]
}
</output_schema>

JSON만 출력하세요. 다른 텍스트는 포함하지 마세요.
```

**사용법:** `<instructions>`, `<reviews>`, `<output_schema>` 같은 XML 태그로 영역을 분리하면 모델이 각 부분의 역할을 명확히 구분해요. JSON 스키마를 정의하면 후처리 코드가 안정적으로 파싱할 수 있어요. 이 패턴은 API 연동, 자동화 파이프라인, 도구 호출에 특히 유용해요.

**예상 결과:** 지정한 JSON 스키마에 맞는 구조화된 분석 결과가 나와요.
