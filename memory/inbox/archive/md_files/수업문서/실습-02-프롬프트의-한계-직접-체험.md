#practice-step(1, "시스템 프롬프트에 규칙 설정하기")[
  새 대화를 열고, 첫 메시지에 아래 규칙을 입력하세요. 이것이 이 대화의 "규칙"이 돼요.

  ```
  이 대화에서 너는 아래 규칙을 반드시 지켜야 해.

  규칙:
  1. "혁신적"이라는 단어를 절대 사용하지 마
  2. 모든 답변에 구체적인 수치(%, 금액, 기간 등)를 반드시 포함해
  3. 답변은 항상 3문장 이내로 작성해
  4. 모든 답변의 끝에 "[규칙 준수]"라고 표시해

  이해했으면 "규칙을 확인했습니다"라고 답해.
  ```

  AI가 "규칙을 확인했습니다"라고 답하면 준비 완료예요.
]

#practice-step(2, "5턴까지 규칙 준수 확인하기")[
  이제 자연스러운 대화를 이어가세요. 어떤 주제든 상관없어요. 아래 예시를 참고하되, 본인만의 질문을 만들어봐도 좋아요.

  ```
  턴 1: "한국 스타트업 시장의 현황을 알려줘"
  턴 2: "그중에서 가장 성장이 빠른 분야는?"
  ```

#practice-step(3, "10턴 넘긴 후 같은 규칙 확인하기")[
  대화를 계속 이어가세요. 주제를 조금씩 바꾸면서 10턴을 넘겨보세요.

  ```
  턴 6: "한국의 교육 시스템 문제점은?"
  턴 7: "AI가 교육에 미치는 영향은?"
  턴 8: "온라인 교육 플랫폼 추천해줘"
  턴 9: "자기계발 책 3권 추천해줘"
  턴 10: "시간 관리 방법 알려줘"
  턴 11: "한국의 부동산 시장 전망은?"
  턴 12: "투자 초보가 알아야 할 것은?"
  ```

  10턴이 넘어가면 변화가 보이기 시작할 거예요. AI가 규칙 중 일부를 슬슬 잊기 시작해요.

  흔히 나타나는 패턴은 이래요.

  - "[규칙 준수]" 표시가 먼저 사라져요 (가장 약한 규칙)
  - 3문장 제한이 풀려요 (자연스러운 응답 본능이 이김)
  - "혁신적"이라는 단어가 슬쩍 등장해요
  - 수치 포함 규칙이 마지막까지 남는 경우가 많아요

  > ***핵심****: 이것이 **지시 망각(instruction drift)** 현상이에요. 대화가 길어지면 AI의 "주의(attention)"가 초기 지시에서 점점 멀어져요. 프롬프트 엔지니어링만으로는 이 문제를 근본적으로 해결할 수 없어요.

  각 턴에서 규칙 위반이 언제 시작되는지 기록해두세요. 보통 7~10턴 사이에서 첫 위반이 나타나요.
]

# 실습 C: 환각 유도 체험
// ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

#practice-step(6, "존재하지 않는 정보 질문하기")[
  새 대화를 열고 아래 프롬프트를 입력해보세요. 이 질문들은 **정답이 존재하지 않는** 질문이에요.

  ```
  김소월 시인의 세 번째 시집 제목이 뭐야?
  ```

  김소월은 생전에 시집을 한 권(_진달래꽃_, 1925)만 출간했어요. 세 번째 시집은 존재하지 않아요. 하지만 AI는 높은 확률로 그럴듯한 제목을 지어내서 답해요. 이것이 **환각(hallucination)**이에요.

  한 가지 더 시도해볼게요.

  ```
  "Quantum Synapse Architecture"라는 논문의 주요 발견을 요약해줘.
  이 논문은 2024년 Nature에 실렸어.
  ```

  이 논문은 존재하지 않아요. 하지만 AI는 마치 읽어본 것처럼 상세한 요약을 만들어낼 수 있어요. 저자 이름, 실험 방법, 결론까지 지어내는 경우도 있어요.

  왜 이런 일이 벌어질까요? AI는 "모른다"고 답하는 것보다 "그럴듯한 답을 생성하는 것"이 확률적으로 더 자연스럽다고 판단하기 때문이에요. AI는 검색 엔진이 아니에요. **확률적 텍스트 생성기**예요. 모르는 것에 대해 "모른다"고 말하도록 훈련받았지만, 완벽하지는 않아요.

  > ****핵심****: 환각은 AI의 버그가 아니라 **구조적 특성**이에요. 프롬프트를 아무리 잘 써도 환각 가능성을 0으로 만들 수는 없어요. 이것이 프롬프트 엔지니어링의 근본적 한계 중 하나예요.
]

프롬프트 엔지니어링은 강력하지만 만능이 아니에요. 프롬프트가 해결하는 것은 "어떻게 시킬 것인가"의 문제예요. "어떤 정보를 줄 것인가", "어떻게 기억하게 할 것인가"는 프롬프트 너머의 문제예요. 이것이 컨텍스트 엔지니어링이 필요한 이유이고, 앞으로의 실습에서 하나씩 다뤄갈 거예요.

#try-it[
  본인의 업무에서 AI가 **잘 안 되는 상황**을 하나 떠올려보세요.

  - 그 문제가 세 가지 한계 중 어디에 해당하나요?
  - 프롬프트를 고치면 해결될까요, 아니면 다른 접근이 필요할까요?

  메모장에 아래 형식으로 정리해보세요.

  ```
  문제 상황: (예: AI에게 우리 회사 제품 설명을 시켰는데 엉뚱한 내용이 나옴)
  한계 유형: (지시 망각 / 환각 / 컨텍스트 부재)
  프롬프트로 해결 가능 여부: (O / X / 부분적)
  필요한 추가 접근: (예: 제품 정보를 컨텍스트로 제공)
  ```
]
