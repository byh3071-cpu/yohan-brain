## AI 문서 작성의 원칙

AI에게 문서 작성을 시킬 때 기억해야 할 원칙이 세 가지 있어요.

#table(
  columns: (auto, 1fr),
  fill: (x, y) => if y == 0 { practice-color.primary } else if calc.rem(y, 2) == 1 { practice-color.bg } else { color.white },
  stroke: 0.5pt + practice-color.border,
  inset: 10pt,
  table.header(
    text(fill: color.white, weight: "bold")[원칙],
    text(fill: color.white, weight: "bold")[설명],
  ),
  text(weight: "bold", fill: practice-color.primary)[초안은 AI, 최종은 사람], [AI가 만든 문서를 그대로 보내면 안 돼요. 반드시 검토와 수정을 거쳐야 해요],
  text(weight: "bold", fill: practice-color.primary)[재료를 먼저 줘야 해요], [AI는 없는 정보를 지어내요. 원본 데이터나 맥락을 먼저 제공해야 정확한 문서가 나와요],
  text(weight: "bold", fill: practice-color.primary)[한 번에 하나씩], [요약, 메일, 회의록을 한 프롬프트에 다 시키지 마세요. 하나씩 시켜야 품질이 올라가요],
)

이제 문서 3종을 하나씩 만들어볼게요.

// ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
# 문서 1: 보고서 요약
// ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

#practice-step(1, "긴 텍스트 붙여넣고 3줄 요약 요청하기")[
  아래는 실습용 보고서 텍스트예요. 실제 업무 보고서가 있다면 그것을 써도 좋아요. (단, 사내 기밀이 포함된 문서는 무료 AI에 붙여넣지 마세요.)

  ```
  [실습용 보고서 텍스트]

  2026년 1분기 마케팅 성과 보고

  1. 전체 요약
  1분기 마케팅 총 지출은 4,500만 원으로 전년 동기 대비 15% 증가했다.
  신규 고객 유치는 1,200명으로 목표(1,000명) 대비 120% 달성했다.
  고객 획득 비용(CAC)은 37,500원으로 전분기 대비 8% 감소했다.

  2. 채널별 성과
  - 검색 광고: 지출 2,000만 원, 신규 고객 600명, CAC 33,333원
  - SNS 광고: 지출 1,500만 원, 신규 고객 350명, CAC 42,857원
  - 콘텐츠 마케팅: 지출 500만 원, 신규 고객 150명, CAC 33,333원
  - 이메일 마케팅: 지출 200만 원, 신규 고객 80명, CAC 25,000원
  - 오프라인 행사: 지출 300만 원, 신규 고객 20명, CAC 150,000원

  3. 주요 이슈
  - SNS 광고의 CAC가 전분기 대비 12% 상승, 크리에이티브 피로도 의심
  - 이메일 마케팅이 CAC 최저, 확대 검토 필요
  - 오프라인 행사 ROI가 현저히 낮음, 축소 또는 형태 변경 필요

  4. 2분기 계획
  - 검색 광고 유지, SNS 크리에이티브 리뉴얼
  - 이메일 마케팅 예산 50% 증액
  - 오프라인 행사 중단, 웨비나로 전환
  ```

  이 텍스트를 AI에 붙여넣고 아래 프롬프트를 추가하세요.

  ```
  위 보고서를 읽고 아래 형식으로 요약해줘.

  형식:
  - 3줄 요약 (각 줄에 핵심 수치 포함)
  - 핵심 인사이트 1개 (데이터에서 도출)
  - 의사결정 포인트 1개 (경영진이 바로 판단할 수 있게)

  독자: 마케팅 예산을 승인하는 경영진
  톤: 간결하고 데이터 중심
  ```

  AI의 결과를 확인해보세요. 3줄 요약에 핵심 수치가 포함돼 있나요? 인사이트가 데이터에서 도출된 것인가요?

  결과가 만족스럽지 않다면 아래를 시도해보세요.
  - "SNS 광고 CAC 상승에 초점을 맞춰서 다시 요약해줘"
  - "2분기 계획의 근거를 1분기 수치와 연결해줘"

  이렇게 한 번의 요약 후 추가 지시를 주는 것을 **반복 다듬기(iterative refinement)**라고 해요. AI 문서 작성에서 가장 효과적인 방법이에요.
]

// ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
# 문서 2: 업무 메일
// ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

#practice-step(2, "상황 컨텍스트 주고 메일 작성하기")[
  새 대화를 열고 아래 프롬프트를 입력하세요.

  ```
  <context>
  발신: B2B SaaS 회사의 영업 담당자
  수신: A사 김 부장님 (지난주 제품 데모 미팅 참석)
  상황: 데모에서 "팀 협업 기능"과 "자동 보고서 생성"에 관심을 보임.
        가격에 대해서는 "내부 검토가 필요하다"고 함.
        이번 주 금요일까지 회신 받기로 했고, 지금은 수요일.
  </context>

  <task>
  김 부장님에게 보낼 후속 메일을 작성해주세요.
  - 데모에서 관심을 보인 기능을 자연스럽게 언급해주세요
  - 금요일 회신을 부드럽게 리마인드해주세요
  - 추가 자료(사례 소개서)를 첨부한다고 언급해주세요
  </task>

  <format>
  - 제목 포함
  - 200자 내외
  - 정중하되 영업 냄새가 나지 않는 톤
  - "안녕하세요, 김 부장님"으로 시작
  </format>
  ```

  결과를 확인해보세요. 아래 항목을 체크하세요.

  - 데모에서 관심을 보인 기능이 언급돼 있나요?
  - 금요일 리마인드가 부드럽게 들어가 있나요?
  - 영업 냄새가 나지 않나요?
  - 바로 복사해서 보낼 수 있는 수준인가요?

  수정이 필요하면 구체적으로 요청하세요. "마지막 문장을 좀 더 부드럽게 바꿔줘", "가격 이야기는 빼줘" 같은 식이에요.
]

// ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
# 문서 3: 회의록 정리
// ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

#practice-step(3, "회의 내용에서 액션아이템 추출하기")[
  새 대화를 열고 아래 실습용 회의 내용을 붙여넣으세요.

  ```
  [실습용 회의 내용, 날것 그대로의 메모]

  2026년 4월 7일 주간 회의

  참석: 이팀장, 김대리, 박사원, 정인턴

  이팀장: 지난주 출시한 신기능 피드백이 들어오고 있어. 김대리가 정리해줘.
  김대리: 네. 긍정 피드백이 70% 정도인데 UI가 복잡하다는 불만이 반복돼요.
  특히 설정 메뉴 접근이 3단계인데 고객들이 못 찾아요.
  이팀장: 설정 메뉴 접근을 2단계로 줄이는 게 가능해?
  박사원: 기술적으로 가능한데 다음 스프린트에 넣으면 2주 걸려요.
  이팀장: 그럼 다음 스프린트에 넣자. 김대리가 요구사항 정리하고
  박사원이 기술 검토. 이번 주 금요일까지.
  정인턴: 고객 인터뷰 5건 잡았는데 일정 조율 중이에요.
  이팀장: 다음 주 수요일까지 완료해줘. 결과는 김대리한테 공유하고.
  김대리: 경쟁사 A가 비슷한 기능을 출시했어요. 분석이 필요할 것 같아요.
  이팀장: 정인턴이 경쟁사 분석 자료 만들어줘. 다음 주 회의에서 발표.
  ```

  이 텍스트를 붙여넣고 아래 프롬프트를 추가하세요.

  ```
  위 회의 내용을 아래 형식으로 정리해줘.

  형식:
  1. 회의 요약 (3줄)
  2. 결정 사항 (번호 목록)
  3. 액션아이템 테이블 (담당자 | 내용 | 마감일)
  4. 다음 회의에서 확인할 사항

  주의: 회의에서 명시적으로 언급된 내용만 포함해.
  추측이나 추가 제안은 하지 마.
  ```

  결과에서 특히 액션아이템 테이블을 확인하세요. 담당자, 내용, 마감일이 원본 회의 내용과 일치하나요?

  > ***핵심****: "추측하지 마"라는 제약조건이 회의록 정리에서 특히 중요해요. 이 제약이 없으면 AI는 "추가로 이런 것도 하면 좋겠습니다"를 덧붙이는 경향이 있어요. 회의록은 사실 기록이지 제안서가 아니거든요.
]

// ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
# 결과물 검토와 수정
// ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

#practice-step(4, "세 가지 결과물 검토하기")[
  세 가지 문서(보고서 요약, 메일, 회의록)를 모두 만들었으면 검토 시간이에요. 아래 체크리스트로 각 문서를 점검해보세요.

  #table(
    columns: (auto, 1fr, 1fr, 1fr),
    fill: (x, y) => if y == 0 { practice-color.primary } else if calc.rem(y, 2) == 1 { practice-color.bg } else { color.white },
    stroke: 0.5pt + practice-color.border,
    inset: 10pt,
    table.header(
      text(fill: color.white, weight: "bold")[검토 항목],
      text(fill: color.white, weight: "bold")[보고서 요약],
      text(fill: color.white, weight: "bold")[업무 메일],
      text(fill: color.white, weight: "bold")[회의록],
    ),
    text(weight: "bold", fill: practice-color.primary)[사실 오류가 없는가], [], [], [],
    text(weight: "bold", fill: practice-color.primary)[톤이 적절한가], [], [], [],
    text(weight: "bold", fill: practice-color.primary)[형식이 맞는가], [], [], [],
    text(weight: "bold", fill: practice-color.primary)[바로 사용 가능한가], [], [], [],
    text(weight: "bold", fill: practice-color.primary)[AI가 지어낸 내용이 있는가], [], [], [],
  )

  문제가 발견되면 AI에게 구체적으로 수정을 요청하세요. "전체적으로 다시 써줘"보다 "두 번째 문단의 톤을 좀 더 부드럽게 바꿔줘"가 훨씬 효과적이에요.

  > ****핵심****: AI 문서 작성의 실전 워크플로우: **재료 제공 → 초안 생성 → 검토 → 구체적 수정 요청 → 최종 확인**. 이 다섯 단계가 하나의 사이클이에요. AI에게 한 번에 완벽한 문서를 기대하지 마세요. 반복 다듬기가 핵심이에요.
]

#try-it[
  지금까지 만든 세 가지 프롬프트를 **템플릿**으로 저장해보세요. 메모장이나 노트 앱에 아래 형식으로 정리하면 앞으로 재사용할 수 있어요.

  ```
  [보고서 요약 템플릿]
  <context> 입력: (보고서 붙여넣기) / 독자: ~ </context>
  <task> 아래 형식으로 요약해주세요 </task>
  <format> 3줄 요약 + 인사이트 1개 + 의사결정 포인트 1개 / 톤: ~ </format>

  [메일 작성 템플릿]
  <context> 발신: ~ / 수신: ~ / 상황: ~ / 관계: ~ </context>
  <task> 위 상황에서 메일을 작성해주세요 </task>
  <format> 제목 포함, ~자 내외, 톤: ~ </format>

  [회의록 정리 템플릿]
  <context> (여기에 회의 내용 붙여넣기) </context>
  <task> 회의록을 정리해주세요 </task>
  <format> 요약 + 결정사항 + 액션아이템 + 다음 확인사항 /
  회의에서 명시적으로 언급된 내용만 포함, 추측 금지 </format>
  ```

  본인의 업무 상황에 맞게 역할, 톤, 형식을 수정해서 자기만의 템플릿을 만들어보세요.
]

#troubleshoot((
  (
    q: "보고서 요약에서 원본에 없는 수치가 등장해요.",
    a: "이것이 바로 환각이에요. '원본에 있는 수치만 사용해'라는 제약조건을 프롬프트에 추가하세요. 그래도 발생하면 반드시 원본과 대조 확인하세요.",
  ),
  (
    q: "메일 톤이 너무 딱딱하거나 너무 가벼워요.",
    a: "톤 조절은 구체적인 예시가 효과적이에요. '비즈니스 캐주얼 톤, ~씨에게 보내는 것처럼'이라고 하거나, 원하는 톤의 메일 예시 한 개를 참고자료로 붙여넣으세요.",
  ),
  (
    q: "회의록에서 AI가 추가 제안을 자꾸 덧붙여요.",
    a: "'회의에서 명시적으로 언급된 내용만 포함해. 추가 제안이나 의견은 절대 넣지 마.'라고 강하게 지시하세요. 그래도 넣는다면 '방금 결과에서 추가 제안 부분만 삭제해줘'라고 후속 지시를 보내세요.",
  ),
