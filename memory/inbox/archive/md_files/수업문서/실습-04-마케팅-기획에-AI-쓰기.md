지난 시간에 보고서 요약, 메일, 회의록 같은 업무 문서를 AI로 만들어봤어요. 오늘은 한 걸음 더 나가서 마케팅과 기획 영역으로 확장해볼게요.

마케팅과 기획은 "정답이 없는 영역"이에요. 그래서 AI가 특히 유용해요. 사람이 방향을 잡고, AI가 다양한 변형을 빠르게 만들어주는 구조가 잘 맞거든요. 오늘은 SNS 캡션, 경쟁사 비교표, 사용자 페르소나, PRD 초안까지 네 가지를 만들어볼 거예요.

# 산출물 1: SNS 캡션 5종 생성
// ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

#practice-step(1, "톤별 SNS 캡션 만들기")[
  같은 제품을 홍보하더라도 톤에 따라 캡션이 완전히 달라져요. 이걸 직접 체험해볼게요.

  새 대화를 열고 아래 프롬프트를 입력하세요.

  ```
  <context>
  플랫폼: 인스타그램
  제품: 프로젝트 관리 앱 "TaskFlow"
  특징: 팀원 간 실시간 협업, 자동 일정 조율, 진행률 대시보드
  타겟: 20~30대 스타트업 직원
  가격: 월 9,900원 (팀당)
  </context>

  <task>
  아래 5가지 톤으로 인스타그램 캡션을 각각 1개씩 작성해주세요.

  톤 1: 유머러스 (밈 느낌)
  톤 2: 전문적 (B2B)
  톤 3: 감성적 (워라밸 강조)
  톤 4: 긴급성 (한정 이벤트)
  톤 5: 사용자 후기 스타일
  </task>

  <format>
  각 캡션은 3줄 이내, 해시태그 3개 포함.
  </format>
  ```

  결과를 보면 같은 제품인데 톤에 따라 완전히 다른 글이 나올 거예요. 이것이 AI를 마케팅에 쓰는 가장 큰 장점이에요. 사람이 5가지 톤을 모두 쓰려면 상당한 시간이 걸리지만, AI는 한 번의 요청으로 다양한 변형을 즉시 만들어줘요.

  > ***핵심****: AI 마케팅 활용의 핵심: **사람이 방향(톤, 타겟)을 잡고, AI가 변형(variations)을 빠르게 생성**하는 것이에요. 100% 완성본을 기대하지 말고, 좋은 초안 후보를 빠르게 여러 개 얻는 도구로 활용하세요.

  5개 캡션 중 마음에 드는 것을 하나 골라 다듬어보세요. "톤 3의 캡션에서 가격 정보를 자연스럽게 추가해줘"처럼 구체적으로 요청하면 돼요.
]

// ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
# 산출물 2: 경쟁사 비교표
// ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

#practice-step(2, "경쟁사 비교표 작성하기")[
  경쟁사 분석은 기획의 기본이에요. AI에게 비교표를 만들어달라고 해볼게요.

  새 대화를 열고 아래 프롬프트를 입력하세요.

  ```
  <context>
  분석 목적: SaaS 프로젝트 관리 시장에서 우리 제품의 포지셔닝 파악

  우리 제품:
  - 이름: TaskFlow
  - 카테고리: 프로젝트 관리
  - 가격: 월 9,900원/팀
  - 주요 기능: 실시간 협업, 자동 일정 조율, 진행률 대시보드
  - 강점: 한국어 최적화, 국내 세금계산서 발행
  - 약점: 글로벌 인지도 없음, 영어 UI 미지원

  경쟁사: Notion, Monday.com, Asana
  </context>

  <task>
  4개 제품(우리 포함)의 비교표를 만들어주세요.
  - 비교 항목: 가격, 주요 기능, 한국어 지원, 타겟 시장, 강점, 약점
  - 비교표 아래에 "우리 제품의 차별화 포인트" 2~3줄 추가
  </task>

  <format>
  - 표 형식으로 작성
  - 확실하지 않은 정보는 "(확인 필요)"라고 표시해주세요
    (경쟁사 가격이나 기능은 수시로 바뀌므로)
  </format>
  ```

  여기서 중요한 것은 마지막 주의사항이에요. 경쟁사의 가격이나 기능은 수시로 바뀌니까 AI가 알고 있는 정보가 정확하지 않을 수 있어요. "(확인 필요)" 표시를 요청하면 AI가 확신 없는 정보를 표시해줘서, 나중에 사람이 확인할 수 있어요.

  결과를 받으면 반드시 경쟁사 웹사이트에서 가격과 기능을 직접 확인하세요. AI가 제공한 비교표는 **초안**이지 **최종본**이 아니에요.
]

// ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
# 산출물 3: 사용자 페르소나
// ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

#practice-step(3, "타겟 사용자 페르소나 만들기")[
  페르소나는 타겟 사용자를 구체적인 인물로 형상화한 것이에요. 마케팅 전략, 제품 기획, 콘텐츠 제작의 기준점이 되어요.

  새 대화를 열고 아래 프롬프트를 입력하세요.

  ```
  <context>
  제품: 프로젝트 관리 앱 "TaskFlow"
  타겟: 스타트업에서 일하는 20~30대 직장인
  목적: 마케팅 전략과 제품 기획의 기준점이 될 페르소나 작성
  </context>

  <task>
  아래 구조로 사용자 페르소나 2명을 만들어주세요.

  각 페르소나 구조:
  1. 이름, 나이, 직업
  2. 하루 일과 요약 (3줄)
  3. 업무에서 느끼는 불만 3가지 (Pain Points)
  4. 프로젝트 관리 도구에 기대하는 것 3가지 (Needs)
  5. TaskFlow를 처음 접했을 때 할 만한 질문 2가지
  6. 이 사람을 설득할 수 있는 한 줄 카피

  </task>

  <format>
  - 두 페르소나는 서로 다른 직군이어야 합니다 (예: 개발자 + 마케터)
  - 현실적이고 구체적인 디테일을 넣어주세요
  - 너무 이상적인 사용자가 아니라, 실제로 있을 법한 사람으로 만들어주세요
  </format>
  ```

  결과를 보면 꽤 구체적인 인물이 나올 거예요. 하지만 여기서도 검토가 필요해요.

  페르소나 검토 포인트:
  - 이 사람이 실제로 존재할 것 같나요?
  - Pain Points가 현실적인가요? (본인이나 주변 사람의 경험과 비교해보세요)
  - 설득 카피가 이 사람에게 실제로 통할 것 같나요?

  마음에 들지 않는 부분이 있다면 "첫 번째 페르소나의 Pain Points를 좀 더 구체적으로 바꿔줘. 예를 들어 '슬랙 알림이 너무 많아서 중요한 메시지를 놓치는 상황' 같은 디테일을 넣어줘"라고 수정 요청하세요.
]

// ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
# 산출물 4: PRD 초안
// ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

#practice-step(4, "PRD 초안 작성하기")[
  PRD(Product Requirements Document)는 제품의 요구사항을 정리한 기획 문서예요. 보통 PM(Product Manager)이 작성하는데, AI가 초안을 잡아주면 시간을 크게 절약할 수 있어요.

  새 대화를 열고 아래 프롬프트를 입력하세요.

  ```
  <context>
  제품: 프로젝트 관리 앱 "TaskFlow"
  신규 기능 요청: "AI 기반 자동 업무 분배"
  현재 상황: 팀장들이 매주 월요일 수동으로 업무를 배분, 평균 2시간 소요
  목표: 업무 분배 시간을 30분 이내로 줄이기
  문서 목적: 이 PRD는 엔지니어·디자이너와 공유할 기획 문서입니다
  </context>

  <task>
  아래 구조로 PRD 초안을 작성해주세요.

  PRD 구조:
  1. 개요 (3줄: 무엇을, 왜, 누구를 위해)
  2. 문제 정의 (현재 상황의 문제점 3가지)
  3. 제안 솔루션 (기능 설명, 작동 방식)
  4. 사용자 시나리오 (Before vs After 비교)
  5. 성공 지표 (측정 가능한 KPI 3가지)
  6. 범위 외 (이번에 다루지 않는 것)
  7. 일정 추정 (개발 소요 기간)
  </task>

  <format>
  - 기술 구현 상세는 제외 (엔지니어가 별도로 작성하기 때문)
  - 각 섹션은 간결하게, 총 A4 1장 분량
  - 실현 가능한 수준으로 작성 (AI가 만능이라는 전제 금지)
  </format>
  ```

  PRD는 다른 산출물보다 검토가 더 중요해요. AI는 기술적 현실성을 정확히 판단하기 어렵거든요. 결과를 받으면 아래 질문으로 검토하세요.

  - 문제 정의가 실제 사용자의 문제와 일치하나요?
  - 제안 솔루션이 기술적으로 실현 가능한가요?
  - 성공 지표가 측정 가능한가요?
  - "범위 외"에 빠진 항목이 있나요?

  > ****핵심****: PRD에서 AI의 가치는 **구조를 잡아주는 것**이에요. 내용의 정확성은 사람이 검증해야 해요. 특히 일정 추정과 기술적 제약은 엔지니어와 반드시 확인하세요.
]

#try-it[
  본인이 관심 있는 제품이나 서비스를 하나 정해서, 오늘 만든 4가지 산출물 중 하나를 직접 만들어보세요.

  추천 순서:
  + 관심 제품의 기본 정보를 정리해요 (이름, 특징, 타겟, 가격 등)
  + 위 Step 1~4의 프롬프트 중 하나를 골라요
  + 프롬프트의 제품 정보를 본인의 제품으로 바꿔요

  + 마음에 안 드는 부분을 구체적으로 수정 요청해요

  실제 업무에서 쓰는 제품이면 더 좋아요. 결과물이 바로 활용 가능할 수 있거든요.

  + **SNS 캡션**: 같은 제품도 톤에 따라 완전히 다른 글이 돼요. AI의 강점은 다양한 변형을 빠르게 만들어주는 것이에요.

  + **PRD 초안*: AI는 구조를 잡아주고, 사람은 내용을 검증해요. 기술적 실현 가능성과 일정은 엔지니어와 확인하세요.
