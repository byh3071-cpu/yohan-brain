## 컨텍스트 엔지니어링이란

프롬프트 엔지니어링은 "어떻게 시킬 것인가"의 문제예요. 컨텍스트 엔지니어링은 "어떤 정보를 줄 것인가"의 문제예요.

이 차이를 식당 비유로 다시 생각해볼게요. 프롬프트 엔지니어링은 **주문서 작성법**이에요. "매운 거 빼고, 크림 파스타, 중간 사이즈"라고 적는 거죠. 컨텍스트 엔지니어링은 **셰프에게 주는 식재료 세트**예요. 같은 주문서라도 냉장고에 뭐가 있느냐에 따라 요리가 달라지잖아요.

AI에게 제공하는 컨텍스트는 크게 세 층으로 나눌 수 있어요.

## 패턴 1: 페르소나 + 규칙 + 예시

가장 효과적인 컨텍스트 설계 패턴은 세 요소를 조합하는 거예요.

```
[페르소나]
너는 ~이야. ~한 경험이 있고, ~한 스타일로 일해.

[규칙]
1. 항상 ~해
2. 절대 ~하지 마
3. ~할 때는 ~형식으로

[예시]
좋은 출력 예시:
"~~~~~~~~~"

나쁜 출력 예시:
"~~~~~~~~~"
```

예시를 포함하면 AI가 원하는 스타일을 훨씬 정확하게 따라해요. "이런 식으로 써줘"라고 말로 설명하는 것보다 실제 예시를 보여주는 게 10배 효과적이에요.

#try-it[
  본인의 업무에 맞는 "페르소나 + 규칙 + 예시" 컨텍스트를 만들어보세요.

  예를 들어:
  - **고객 응대 봇**: CS 전문가 페르소나 + 답변 규칙(길이, 톤) + 좋은/나쁜 답변 예시
  - **보고서 작성 도우미**: 업계 전문가 페르소나 + 형식 규칙 + 기존 보고서 예시
  - **콘텐츠 작성 도우미**: 브랜드 보이스 페르소나 + 글쓰기 규칙 + 기존 콘텐츠 예시

  작성한 컨텍스트를 AI에 입력하고, 같은 질문을 컨텍스트 유/무로 비교해보세요.
]

## 패턴 2: 대화 기록 관리

실습 2차시에서 대화가 길어지면 AI가 지시를 잊는다는 걸 확인했어요. 이에 대한 대응 전략이 두 가지 있어요.

#table(
  columns: (auto, 1fr, 1fr),
  fill: (x, y) => if y == 0 { practice-color.primary } else if calc.rem(y, 2) == 1 { practice-color.bg } else { color.white },
  stroke: 0.5pt + practice-color.border,
  inset: 10pt,
  table.header(
    text(fill: color.white, weight: "bold")[전략],
    text(fill: color.white, weight: "bold")[방법],
    text(fill: color.white, weight: "bold")[적합한 상황],
  ),
  text(weight: "bold", fill: practice-color.primary)[새 대화 시작], [중요 규칙을 첫 메시지에 다시 넣고 새 대화를 시작], [규칙이 절대 바뀌면 안 되는 경우],
  text(weight: "bold", fill: practice-color.primary)[중간 요약 삽입], [대화 중간에 "지금까지의 대화를 요약해줘" 후 새 대화에 요약만 넣기], [긴 대화를 이어가야 하는 경우],
)

두 번째 전략을 직접 해볼게요.

```
지금까지 우리가 논의한 내용을 아래 형식으로 요약해줘.

1. 결정된 사항 (번호 목록)
2. 아직 결정되지 않은 사항
3. 다음에 논의할 주제

이 요약은 새 대화에서 컨텍스트로 사용할 거야.
```

이렇게 AI에게 요약을 만들게 한 뒤, 새 대화를 열어 그 요약을 첫 메시지에 붙여넣으면 돼요. 대화의 핵심만 넘기고 불필요한 잡담은 버리는 셈이에요. 이게 "대화 기록 관리"의 기본이에요.
