#summary-box(title: "배경지식: 에이전트 루프")[
  이론편에서 에이전트 루프(인식 → 계획 → 실행 → 관찰)를 배웠어요. Claude Code는 이 루프를 자동으로 수행하는 에이전트예요.

  작은 작업에서는 루프가 한두 번이면 끝나요. "함수 하나 작성해줘"라고 하면 한 번의 루프로 충분하죠. 하지만 큰 작업에서는 루프가 수십 번 반복돼요. 파일을 분석하고, 수정하고, 테스트하고, 실패하면 다시 분석하고, 이 과정이 자동으로 돌아가요.

  이 루프를 더 구체적으로 표현하면 **Gather → Act → Verify** 세 단계예요.

  - **Gather**: 정보 수집 (파일 읽기, 구조 분석)
  - **Act**: 행동 수행 (코드 수정, 파일 생성)
  - **Verify**: 결과 검증 (테스트 실행, 빌드 확인)
]

## 테스트를 일부러 실패시켜보기

자동 수정 루프를 더 명확하게 관찰하기 위해, 일부러 테스트를 실패시켜볼게요.

```
> tests/test_main.py에 일부러 실패하는 테스트를 2개 추가해줘.
> 하나는 존재하지 않는 함수를 호출하고,
> 다른 하나는 잘못된 반환값을 기대하는 테스트야.
> 그런 다음 모든 테스트가 통과하도록 수정해줘.
```

Claude가 이 요청을 처리하는 과정을 지켜보세요.

1. 먼저 실패하는 테스트를 추가해요
2. pytest를 실행해서 실패를 확인해요
3. 실패 원인을 분석해요 (함수가 없다, 반환값이 다르다)
4. 원인에 맞게 코드를 수정해요 (함수 추가, 로직 수정)
5. 다시 pytest를 실행해요
6. 모두 통과하면 완료를 보고해요

이 과정 전체가 자동이에요. 여러분은 지켜보기만 하면 돼요.

> ****핵심****: 에이전트 루프의 핵심 가치는 "실패를 자동으로 복구하는 능력"이에요. 사람이라면 에러를 보고 → 원인을 분석하고 → 수정하고 → 다시 실행하는 이 과정을 수동으로 해야 하지만, 에이전트는 이걸 자동으로 반복해요.

### Gather (정보 수집)

Claude가 행동하기 전에 필요한 정보를 모으는 단계예요.

- 파일 구조 파악 (Glob, Bash)
- 파일 내용 읽기 (Read)
- 패턴 검색 (Grep)
- 의존성 분석 (import문 추적)
- 기존 테스트 확인

여러분이 할 수 있는 것: "먼저 전체를 분석해줘"라고 명시적으로 Gather를 요청하세요.

### Act (행동 수행)

수집한 정보를 바탕으로 실제 변경을 수행하는 단계예요.

- 코드 수정 (Edit, Write)
- 파일 생성/삭제
- 설정 변경

여러분이 할 수 있는 것: 변경 범위와 우선순위를 명확히 지시하세요.

### Verify (결과 검증)

변경이 올바른지 확인하는 단계예요.

- 테스트 실행 (Bash: pytest)
- 빌드 확인 (Bash: ruff check)
- 결과 보고

  - 테스트 실패 시 Claude는 자동으로 원인 분석 → 수정 → 재테스트 루프를 돌아요
  - 에이전트의 자율성은 테스트 커버리지에 비례해서 조절해야 해요
  - CLAUDE.md에 빌드/테스트 명령어를 적어두면 Verify 단계가 자동화돼요
]
