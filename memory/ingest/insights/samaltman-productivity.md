# Sam Altman — Productivity (2018)

[학습 의도 — 추론] 요한 브레인의 "지식 복리" 설계와 같은 원리(복리 성장)를 개인 생산성 차원에서 다룬 1차 소스. 시스템(레버리지 구조) 우선 원칙과 교차 검증할 가치.

## 핵심 요약 (3줄)

- **복리가 핵심:** 작은 생산성 향상도 50년 복리로 누적되면 거대하다 — "매일 1% 개선 + 10% 더 완수"의 복리 격차는 압도적.
- **방향 > 속도:** 무가치한 방향으로 빠르게 움직이는 건 의미 없다. "무엇을 할지 고르는 것"이 생산성의 최우선 요소인데 대부분 무시한다. 생산성 포르노(시스템 최적화 집착)를 경계하고 "올바른 문제인가"를 더 물어라.
- **3기둥 시스템:** ① 중요한 일은 반드시 해낸다 ② 멍청한 일에 시간 낭비하지 않는다 ③ 리스트를 많이 만든다(연/월/일, 종이, 자주 재작성) — 분류·사이징 없이 별표 하나만.

## 핵심 키워드

- 복리 성장(compound growth)의 커리어 적용 · 독립적 사고(마지막 대화자에 동조하면 위험) · 위임 기준(좋아하고 잘하는 사람에게) · 모멘텀 우선순위(완수→기분→더 완수) · 시간 가치(시급 $100인 사람이 $20 아끼려 2시간 소모) · 연 단위 최적 배분(하루가 아니라 1년을 최적화) · 회의 15-20분 또는 2시간(기본 1시간이 낭비 원인) · 물리적 기반(수면>운동>영양, 합치면 1.5x) · 약간의 과커밋(집중 훈련, 과도하면 재앙)

## 인사이트 → 적용

- "올바른 일 선택"에 시간을 확보하는 그의 방법 = 독서·흥미로운 사람·자연 — 입력 다양성이 방향 감각의 원료.
- 동기 실종 1~2주는 누구에게나 온다 — 해결책을 찾기보다 "안개는 걷힌다"는 신뢰 + 기분 망치는 사람·상황 회피.
- 가족·친구·좋아하는 일을 생산성과 맞바꾸지 말 것 — 행복 하락은 순 생산성 손실.

## Yohan OS 적용

- 바로 적용 가능한 것: ① "연 단위 최적 배분" 관점을 goals/ 운영에 — 일일 todo 최적화보다 분기·연 단위 방향 점검 우선. ② 회의·작업 블록 15-20분/2시간 이원화는 자동화 배치·세션 설계에 그대로 적용 가능.
- 도입하면 좋은 것(실험): "리스트 재작성(re-transcribe)" 의식 — inbox/quick-capture를 주기적으로 처음부터 다시 쓰며 항목 추가/제거 판단을 강제 (그의 종이 리스트 원리의 디지털 변형).
