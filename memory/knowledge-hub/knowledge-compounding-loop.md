# 지식 복리 루프 — 왜 이 구조인가 (운영 관점)

## 한 줄 정의

요한 브레인의 지식이 **금융 복리처럼 자가증식**하도록 설계한 운영 루프. 구조 명세는 `docs/KNOWLEDGE-LOOP.md`가 SoT이며, 이 문서는 **왜 그 구조여야 하는지**와 운영하며 확인된 근거를 기록한다.

## 핵심 구조

복리의 원리 (노션 PKM 전략 SUMMARY, 2026-04-14):

> 지식 A + 지식 B → 연결에서 새 지식 C 탄생 → C가 다시 A·B와 연결 → 폭발적 성장.

이 원리가 작동하려면 4개 메커니즘이 모두 살아 있어야 한다:

| 메커니즘 | 없으면 생기는 일 | 루프 단계 |
|----------|----------------|----------|
| 승격 (insight→자산) | 요약이 1회용으로 끝남 — 이자 없는 예금 | ④ |
| 역전파 (새 지식→기존 문서) | 자산이 늙어 신뢰 상실 → 아무도 안 읽음 | ⑤ |
| 트리플 대조 (관계 발견) | A·B가 연결 안 됨 → C가 태어나지 않음 | ⑥ |
| 재사용 (세션 시작 주입) | 매번 재조사 — 원금만 반복 입금 | ⑦ |

## 출처별 관점 (교차검증)

| 소스 | 핵심 주장 | 수렴/충돌 |
|------|----------|----------|
| 노션 PKM 전략 (지식 복리) | 연결이 새 지식을 만든다 — 복리의 정의 | 수렴 |
| Karpathy LLM Wiki 패턴 | 누적은 raw/wiki 분리 + 지속 갱신에서 나옴 | 수렴: 복리의 전제 조건 |
| 현대AI개론 Ch.18 | 한 세션에서 "새 링크·정규 용어·승격 노트"가 늘면 복리와 정합 | 보완: 복리의 측정 지표 제시 |
| Sam Altman — Productivity (2018) | 복리 성장은 금융을 넘어 커리어·생산성에도 작동 — "매일 1% 개선의 50년 복리" | 보완: 복리 원리가 돈→지식→커리어 3도메인에서 반복 확인 [2026-06-12 역전파] |

- 혼동 방지가 복리의 전제: 사람·AI가 어디에 쓸지 헷갈리면 같은 지식이 두 곳에 갈라져 쌓이고, 연결이 끊겨 복리가 멈춘다 → "한 폴더 = 한 역할" 경계가 구조의 1번인 이유.
