# 팔란티어 온톨로지 — 요한 OS의 전략적 원형

## 한 줄 정의

팔란티어의 본질은 AI가 아니라 **문제 해결 능력**이며, 온톨로지(3레이어 유기체)로 문제를 투명화하고 문제 중심 접근으로 기업 고유의 정체성(제로 투 원)을 형성한다 — 자율경영은 수단, **인간+AI 공생이 최종 방향**. (노션 「팔란티어 지식 허브」 시리즈 핵심 명제의 로컬 재구성)

## 핵심 구조

**요한 OS = 팔란티어의 축소판:**

| 팔란티어 | 요한 브레인 대응 |
|----------|------------------|
| 로우데이터 | RESOURCE (`memory/ingest/` 원본) |
| 온톨로지 | SUMMARY + 트리플 맵 (`insights/` + `knowledge-hub/triple-map.md`) |
| 지식그래프 | 위키·허브 크로스링크 네트워크 |

**온톨로지 3레이어 → OS 재해석:**

| 레이어 | 팔란티어 의미 | 요한 OS 대응 |
|--------|--------------|--------------|
| 시멘틱 (의미) | 객체·속성 정의 | DB 속성·frontmatter 스키마 |
| 카이네틱 (동작) | 워크플로우 | 프로토콜 (`memory/rules/`) |
| 다이나믹 (판단) | 의사결정 로직 | AI 에이전트 (노뚝이·Claude) |

**운영 원리 (노션 시리즈 6클러스터·트리플 41건의 수렴점):**

1. **시장 = 문제의 총합** — `solves`/`addresses`/`exposes` 트리플로 "내가 해결하는 문제의 지도"를 구축하면 제품 간 문제 겹침과 다음 방향이 자동 도출된다.
2. **진화형 시스템** — 완벽한 프로토콜을 만들려 하지 말고 실행하며 진화시킨다.
3. **자동화는 수단** — 정체성 있는 지식 체계 구축이 진짜 목표.
4. **AI 환각 = 시스템 진단 신호** — 에이전트가 헛소리하면 프로토콜·DB 불일관성부터 의심한다 (LLM 할루시네이션 caused_by 시스템 불투명성).
