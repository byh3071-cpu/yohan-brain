# PARA Method

## 정의 (1~2문장)
- Projects, Areas, Resources, Archives 4분류로 정보를 관리하는 프레임워크. 지식 관리의 구조적 뼈대로, 폴더 난립을 방지하고 수집과 정리를 분리.

## Verified (소스 기반)
- Karpathy가 기존 사례 조사 + 수일 브레인스토밍 끝에 PARA Method를 채택. [source: karpathy-obsidian-para-workflow]
- 지식·캡처가 쌓이면 정리·재검색·업데이트가 부담 — 이 병목을 PARA로 해결. [source: karpathy-obsidian-para-workflow]
- PARA를 변형한 대분류 약 10개만 유지 — 하위 폴더 난립 금지. [source: obsidian-vault-para-system-guide]
- 수집과 정리 분리: 이동 중엔 인박스에만 던지고 폴더 고민 없이 복귀. [source: obsidian-vault-para-system-guide]
- 저마찰 수집 / 고레버리지 정리 원칙. [source: obsidian-vault-para-system-guide]
- 개인 PKM 네 원칙으로 제시되는 예: 원본 보존, 노트 간 최소 2링크, 주 1회 이상 정리, AI 요약·연결 제안 후 사람이 판단. [source: modern-ai-ch18-knowledge-management-karpathy-wiki]
